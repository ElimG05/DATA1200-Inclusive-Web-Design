GitHub

Samarbeidet på GitHub

I løpet av denne perioden har vi brukt GitHub aktivt for å strukturere, dele og utvikle prosjektet vårt. Vi opprettet et eget repository, der vi har jobbet sammen ved hjelp av GitHubs mange funksjoner som commits, branches og mye annet. Branches har vært en av de mest nyttige funksjonene i GitHub. De har lagt oss jobbe med forskjellige deler av prosjektet uten å påvirke hovedkodebasen. Vi opprettet egne branches for ulike deler, blant annet header, activity og footer, noe som ga oss frihet til å eksperimentere og forbedre koden. Når vi var ferdige med å jobbe i branchen, var det bare å merge det inn i hovedkodebasen.

Commit historikk

Commit-historikken på GitHub har vært en viktig funksjon for oss. Det har gitt oss en detaljert logg over alle endringer. Denne loggen gir en oversikt over hva som ble gjort, når og av hvem. Ved å legge til kommentarer til commits kan vi forklare endringene våre, noe som har hjulpet mye med forståelse i gruppen.

Visuell sporing av endringer

En annen viktig funksjon GitHub har er å kartlegge alle endringene man gjør etter hvert som man committer. Dette gjør det enkelt å spore og analysere endringer gjennom en visuell oversikt. Når man klikker på en commit, får man opp en detaljert visning av hvilke linjer som er lagt til, endret eller slettet.

Fordeler med GitHub

  • Branches: Gir muligheten for å kunne jobbe med separate deler av prosjektet uten å påvirke hovedkoden.
  • Effektiv sporing av kodeendringer: GitHub holder oversikt over alle endringer med versjonshistorikk, slik at vi kan gå tilbake til tidligere versjoner ved behov og ingenting går tapt.
